CSS 방법론

2. BEM

BEM은 Block(블록), Element(요소), Modifier(수정자) 개념을 기반으로 CSS 클래스 이름을 체계적으로 관리하는 방법론이다.

재사용성과 유지보수성을 높이기 위해 개발되었다.

2-1. 블록 (Block)

  • 독립적인 UI 구성요소이며 의미상 스스로 존재할 수 있는 컴포넌트이다.
  • 블록 이름은 의미 있는 명사로 정한다 (기능 또는 역할 중심).
  • 절대 다른 블록의 내부 요소에 의존하지 않아야 한다. (.menu__item은 .menu 블록 안에서만 의미 있음.)

2-2. 요소 (Element)

  • 블록의 구성요소로, 블록에 종속된다.
  • 블록 이름과 두 개의 밑줄(__)로 연결되는 형식을 따른다.
  • 요소는 블록 안에서만 사용해야 하며, 블록 없이 단독으로 쓰지 않는다.
  • 다른 블록의 요소와 중복되지 않도록 블록명을 반드시 앞에 붙여야 한다.

2-3. 수정자 (Modifier)

  • 블록이나 요소의 상태나 변형을 나타낸다. (수정자는 의미 변화가 아닌 상태 변화만 표현해야 함.)
  • 한 개의 밑줄(_)로 상태를 명시한다.
  • block--modifier 또는 block__element--modifier 형식을 사용할 수도 있으나, 공식 BEM에서는 언더스코어 _ 방식을 권장한다.

<div class="menu__item menu__item_selected">Home</div>
